# Translation Files - READ THIS FIRST

**Use the `@i18n-lead` agent when working with translations.**

## Quick Rules

| File | Editable? | Notes |
|------|-----------|-------|
| `en-US.json` | YES | Source of truth - add new strings here |
| `pseudo.json` | NO | Auto-generated by `make translate-pseudo` |
| All other `*.json` | NO | Managed by Crowdin - never edit directly |

## What You Can Do

- **Add new strings:** Edit `en-US.json` only, commit, push to main
- **Fix English text:** Edit `en-US.json`, commit, push to main
- **Generate test locale:** Run `make translate-pseudo`

## What You Cannot Do

- Edit `de-DE.json`, `fr-FR.json`, `es-ES.json`, etc. directly
- Add translations yourself (even if you know the language)
- Copy strings from `en-US.json` to other locales

## Why This Matters

Crowdin is the source of truth for translations. When you edit non-English files directly:
1. Crowdin doesn't know about your changes
2. The next Crowdin sync creates conflicts
3. Your changes may be overwritten or cause merge failures

## Workflow

```
1. Add/edit strings in en-US.json
2. Commit and push to main
3. Crowdin syncs automatically
4. Translators translate in Crowdin
5. Crowdin creates PR (i18n/crowdin-sync â†’ main)
6. Review and merge PR
```

## Need Translations Fast?

```bash
make translate-upload    # Push source to Crowdin
# ... translate in Crowdin ...
make translate-download  # Pull translations
```
